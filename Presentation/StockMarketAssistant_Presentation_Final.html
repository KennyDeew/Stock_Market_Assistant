Проблема и решение

Проблема

  • Нет единой платформы для мониторинга котировок
  • Существующие решения сложные и дорогие
  • Отсутствие удобного многопользовательского доступа

Решение

  • Real-time котировки через WebSocket
  • Простой интерфейс для портфелей
  • Многопользовательский доступ (RBAC)
  • Аналитика и рейтинги активов

Non-Functional Requirements

Производительность

  • WebSocket latency: < 100ms
  • Поддержка 1000+ concurrent users

Масштабируемость

Надежность и безопасность

Надежность

  • Availability: 99.9% SLA
  • Circuit Breaker для защиты
  • Health checks для всех сервисов

Безопасность

  • TLS 1.3, JWT токены
  • Rate limiting
  • Input validation

Security Architecture

Authentication

  • JWT (15 min lifetime)
  • Refresh token rotation
  • ASP.NET Identity

Authorization

  • RBAC (ADMIN, USER)
  • Row-level security
  • Policy-based

API Security

  • Rate limiting (100 req/min), CORS, Input validation
